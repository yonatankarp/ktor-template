# C4 Architecture Diagrams

Source of truth is [`architecture.dsl`](./architecture.dsl) — Structurizr DSL describing the system context, containers, and components.

The PNG renders in this directory are regenerated by the `Generate Docs` GitHub Actions workflow whenever `architecture.dsl` changes. Don't edit the PNGs by hand.

## Levels

- **System Context** — what the system is, who uses it, what external systems it talks to
- **Container** — the runtime processes/services inside the system (the Ktor app, Postgres)
- **Component** — the logical components inside the Ktor app (HTTP adapters, use cases, output adapters)

## Regenerating locally

Use [Structurizr Lite](https://docs.structurizr.com/lite) or [`generate-c4-model-action`](https://github.com/university-of-york/generate-c4-model-action) (what the GH workflow uses).
